## Выполнение работы

### 1. Подготовка лабораторного стенда

Лабораторный стенд был развёрнут на платформе **Oracle VirtualBox** с использованием **Vagrant** для автоматизации создания и настройки виртуальных машин. В качестве базовой операционной системы выбрана **Fedora 39** (образ `fedora/39-cloud-base`).

#### 1.1. Структура стенда

Стенд состоит из двух виртуальных машин:

| Имя машины   | IP-адрес       | Роль                                                  |
|--------------|----------------|-------------------------------------------------------|
| `dns-server` | `192.168.56.10`| Первичный DNS-сервер (BIND9)                          |
| `dns-client` | `192.168.56.20`| Клиентская машина для тестирования разрешения имён    |

Обе машины подключены к **внутренней виртуальной сети** `dns_lab` с типом адаптера **Host-Only**, что обеспечивает изоляцию стенда от внешней сети и стабильность IP-адресации.

#### 1.2. Конфигурация Vagrant

Файл `Vagrantfile` содержит следующую конфигурацию:

- **Провайдер:** VirtualBox
- **Оперативная память:** 1024 МБ для каждой ВМ
- **Количество ядер CPU:** 1
- **Синхронизация папок:** каталог `./provision` монтируется в `/vagrant` внутри ВМ для передачи скриптов автоматизации

#### 1.3. Предварительная настройка сервера

Перед установкой DNS-сервера на машину `dns-server` были выполнены следующие действия:

- Обновление пакетной базы: `sudo dnf update -y`
- Установка сетевых утилит: `sudo dnf install net-tools -y`
- Настройка статического IP-адреса через конфигурационный файл `/etc/sysconfig/network-scripts/ifcfg-enp0s3`
- Отключение SELinux для упрощения отладки (временная мера): `sudo setenforce 0`
- Настройка файервола для разрешения DNS-трафика: `sudo firewall-cmd --add-service=dns --permanent && sudo firewall-cmd --reload`
